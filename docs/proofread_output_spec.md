# 文學／實用書校對輸出規格

本文件整理 proofreadK 在文學、實用書、訓練／健康類書稿校對時的輸出規格。目標是讓 AI 產生可審核的 JSON action，並讓校對員能用 Excel 管理全書口徑、風險和二輪審訂。

## 一、給 AI 的 JSON Action 規則

完整欄位以根目錄的 `ai_output_schema_v1.md` 為準。本節只整理 AI 生成時最重要的執行規則。

### 1. 輸出邊界

- 只輸出 JSON array。
- 不輸出 Markdown、總結、全文改寫或整段重寫。
- 每一條 action 都必須可定位、可審核、可回退。
- 每段最多 5 條建議。
- 不確定時用 `comment`，不要硬改。

### 2. action_type 選擇

| action_type | 用途 | 使用條件 |
| --- | --- | --- |
| `replace` | 精準替換 | 明確錯字、標點、空格、格式，或已確認全書口徑。 |
| `delete` | 精準刪除 | 明確多餘字詞、重複內容、已確認不應保留的殘留標記。 |
| `comment` | 只標示 | 需人工判斷、需查證、涉及作者語氣、術語未定、相片／版面待核。 |

### 3. 分級與動作政策

| grade | grade_label | AI 行為 |
| --- | --- | --- |
| `A` | 可直接改 | 可用 `replace/delete`。 |
| `B` | 建議改 | 可用精準 `replace`；若涉及口徑或語氣，改用 `comment`。 |
| `C` | 只標示 | 使用 `comment`。 |
| `D` | 需人工判斷 | 必須使用 `comment`。 |

### 4. 必須用 comment 的情況

- `original_text` 無法逐字定位。
- 替代詞未定，例如 `骨盆／盆骨`。
- 器材、術語或專名需要全書統一，例如 `拉力帶／橡筋帶／橡皮筋`。
- 涉及作者聲線、比喻、幽默、章題風格、人物口吻。
- 涉及醫療、安全、訓練效果、研究來源或數據。
- 相片標記、圖說、附件、設計稿或排版稿待確認。

### 5. 本書常見 action 例子

| 原文 | 建議 | action_type | 分級 | 原因 |
| --- | --- | --- | --- | --- |
| `6週內` | `6 週內` | `replace` | A 可直接改 | 中數混排需留半形空格。 |
| `很努力?` | `很努力？` | `replace` | A 可直接改 | 中文句子或章題使用全形問號。 |
| `橡筋拉力帶` | 留批註 | `comment` | D 需人工判斷 | 正式器材名未確認。 |
| `attachment: ch10-02` | 留批註 | `comment` | C 只標示 | 這是內部相片位置標記，不是正文。 |
| `血壓飆升` | 留批註 | `comment` | C 只標示 | 涉及安全／醫療風險，不由 AI 查證或改寫。 |

## 二、給校對員的 Excel 欄位與分級指南

Excel 不是單純匯出 AI 建議，而是全書校對管理表。它應協助校對員追蹤批次、統一口徑、二輪審訂、相片標記和風險項目。

### 1. 建議工作表

| 工作表 | 用途 | MVP 狀態 |
| --- | --- | --- |
| 使用說明 | 說明版本、適用範圍、不可硬改原則。 | 建議建立 |
| 已定稿口徑 | 記錄使用者已確認的書名、語言、術語、格式。 | 必須建立 |
| 批次問題清單 | 收錄每批校對問題主表。 | 必須建立 |
| 待二輪審訂 | 給作者、責編或專業審訂確認。 | 必須建立 |
| 專名術語總表 | 管理專名、術語、英文名、譯名。 | 必須建立 |
| 動作名稱表 | 管理運動、技術、訓練動作名稱。 | 視書稿建立 |
| 用字格式總表 | 管理標點、空格、地域詞、統一用字。 | 必須建立 |
| 數字訓練格式 | 管理時間、組數、配速、距離、百分比、單位。 | 實用書建議建立 |
| 分區相片檢查 | 管理 attachment、圖說、相片順序、設計稿待核。 | 有圖書稿必須建立 |
| 灰區人工判斷 | 管理作者聲線、章題、比喻、地域語感。 | 必須建立 |
| 來源安全風險 | 管理醫療、安全、效果、研究來源和數據風險。 | 健康／訓練類必須建立 |
| 敏感表述 | 管理族群、性別、疾病、政治、歷史等敏感表述。 | 視書稿建立 |
| SOP 回灌候選 | 暫存可跨書複用的新規則。 | 暫不急於建立 |

### 2. 批次問題清單欄位

| 欄位 | 說明 |
| --- | --- |
| 批次 | 例如「第一批」「第六批」。 |
| 覆蓋範圍 | 本批章節或分區。 |
| 位置 | 章節、小節、段首字或關鍵詞。無穩定版面時不可填虛構頁碼。 |
| 原文 | 需回查的原文。 |
| 建議處理 | 建議改法或批註方向。 |
| action_type | `replace`、`delete`、`comment`。 |
| 問題類別 | 錯字、術語、標點、數字、分區文本、來源安全、灰區等。 |
| 分級 | A/B/C/D 或中文分級。 |
| 原因 | 具體依據。 |
| 是否涉及全書統一 | 是／否。 |
| 是否加入詞表／附則 | 是／否，並標示應加入哪張表。 |
| 是否需人工判斷 | 是／否。 |
| 是否需來源查證 | 是／否。 |
| 狀態 | 待處理、已接受、已拒絕、待作者、待責編、已定稿。 |
| 備註 | 補充風險、限制或處理口徑。 |

### 3. 專名術語總表欄位

| 欄位 | 說明 |
| --- | --- |
| 詞項 | 原文出現的詞。 |
| 候選主用詞 | 建議統一詞。 |
| 異文 | 其他出現形式。 |
| 英文／原文 | 如 `alignment`、`Spiral Line`。 |
| 所屬類型 | 專名、術語、器材、動作、身體部位、概念。 |
| 首次出現位置 | 章節或段首。 |
| 建議口徑 | 保留、統一、待確認、列例外。 |
| 確認者 | 作者、責編、校對、專業顧問。 |
| 狀態 | 候選、已確認、拒絕、二輪再議。 |
| 備註 | 說明風險或例外。 |

### 4. 用字格式總表欄位

| 欄位 | 說明 |
| --- | --- |
| 項目 | 例如問號、冒號後空格、PB、Q 彈。 |
| 原出現形式 | 稿件中的寫法。 |
| 建議形式 | 建議統一寫法。 |
| 規則類型 | 標點、空格、數字、英文縮寫、地域詞。 |
| 適用範圍 | 全書、某章、圖說、訓練步驟。 |
| 是否可直接改 | 是／否。 |
| 例外 | 哪些語境可保留原寫法。 |
| 狀態 | 候選、已確認、需人工判斷。 |

### 5. 分級指南

| 分級 | 校對員處理方式 | 常見例子 |
| --- | --- | --- |
| A 可直接改 | 可接受 AI action，或直接套用追蹤修訂。 | `6週`、半形問號、中文標點後多空格。 |
| B 建議改 | 先看上下文；若不影響語氣可接受，否則轉批註。 | `八字型／八字形`、英文術語大小寫。 |
| C 只標示 | 不直接改正文，只加入批註或風險表。 | 效果語句、安全提示、章題聯想、相片標記。 |
| D 需人工判斷 | 放入待二輪審訂，不由校對員自行定稿。 | `骨盆／盆骨`、`跑者／跑手`、`alignment` 譯法。 |

### 6. 本次對談萃取出的重點候選項

| 類型 | 候選項 | 建議處理 |
| --- | --- | --- |
| 身體部位 | `盤骨／骨盆／盆骨` | `盤骨` 可疑，但替代詞須作者確認。 |
| 器材 | `拉力帶／橡筋帶／橡皮筋` | 建立正式器材名；比喻用語可列例外。 |
| 讀者稱謂 | `跑者／跑手` | 香港方向可接受「跑手」，但需全書統一。 |
| 技術概念 | `alignment／身體對應／對線` | 核心術語，必須作者或責編定稿。 |
| 身體術語 | `脊柱／脊椎` | 合併圖說與正文異文審訂。 |
| 訓練術語 | `地面接觸時間／觸地時間` | 確認是否同義或分義。 |
| 動作名稱 | `平板支撐／平板撐` | 統一動作名稱。 |
| 運動科學 | `垂直振幅／垂直震幅` | 建議核對主用詞。 |
| 步態術語 | `步長／步幅` | 確認是否同義。 |
| 英文大小寫 | `Spiral Line／Spiral line` | 建議統一為正式名稱。 |

### 7. 成功判斷

一份合格的校對輸出應達到以下效果：

- AI JSON action 可被 parser 驗證。
- Word 追蹤修訂只套用可安全定位的 `replace/delete`。
- 灰區、風險和術語問題保留為批註或 Excel 待確認項。
- 校對員可從 Excel 看出每批進度、未處理範圍、全書統一候選和二輪審訂項目。
- 不把校對變成潤飾、改寫或外部考證。
